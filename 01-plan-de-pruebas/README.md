#  Plan de Pruebas – Sistema Web Demostrativo
### Autor: Leander Oropeza – QA Functional Specialist

---

## 1. Introducción
El objetivo de este plan de pruebas es definir el enfoque, alcance, recursos y actividades necesarias para validar la calidad funcional de la aplicación web seleccionada para este portafolio.

---

## 2. Alcance
**Incluye:**
- Validación funcional de módulos principales (login, registro, CRUD, navegación).
- Verificación de reglas de negocio.
- Validación de mensajes de error.
- Pruebas de regresión básicas.
- Validación en navegadores Chrome/Firefox.

**No incluye:**
- Pruebas de rendimiento.
- Pruebas de accesibilidad.
- Pruebas de seguridad avanzadas.

---

## 3. Objetivos
- Garantizar que el sistema cumple con los requerimientos funcionales.
- Identificar defectos antes de liberar una versión.
- Documentar el comportamiento esperado y obtenido.
- Asegurar la calidad de la experiencia del usuario.

---

## 4. Criterios de entrada
- Requerimientos definidos.
- Ambiente disponible.
- Accesos habilitados.
- Historias de usuario priorizadas.

---

## 5. Criterios de salida
- Defectos críticos resueltos.
- Regresión del sprint aprobada.
- Casos de prueba ejecutados y documentados.
- Evidencias cargadas.

---

## 6. Tipos de pruebas
- Pruebas funcionales
- Pruebas de regresión
- Pruebas exploratorias
- Pruebas de integración básica
- Pruebas End-to-End

---

## 7. Entregables
- Plan de pruebas
- Casos de prueba
- Reporte de defectos
- Matriz de trazabilidad
- Evidencias de ejecución

---

## 8. Herramientas utilizadas
- JIRA / Xray
- Excel
- Postman
- GitHub
- Chrome DevTools

---

## 9. Riesgos
- Cambios de requisitos no comunicados
- Fallos intermitentes de ambiente
- Datos inconsistentes durante la prueba

---

## 10. Recomendaciones
- Mantener trazabilidad entre requisitos y casos.
- Ejecutar regresión completa antes del release.
- Uso periódico de pruebas exploratorias para encontrar defectos no previstos.
