# Portafolio Profesional de QA Funcional  
### Leander Oropeza – Quality Assurance Functional Specialist

Bienvenido a mi portafolio profesional como **QA Funcional**, donde presento mi experiencia real diseñando, ejecutando y gestionando pruebas para aplicaciones web, móviles y sistemas bancarios.

Este repositorio refleja mi capacidad para asegurar la calidad del software mediante prácticas sólidas de testing funcional, exploratorio y de regresión, así como el uso de herramientas modernas de gestión y documentación.

---

##  Sobre mí

Soy QA Funcional con más de **2 años de experiencia**, principalmente en el sector bancario, donde he validado flujos críticos como:

- Transacciones financieras (ingresos y retiros)  
- Cobro e impresión de cheques  
- Conciliaciones contables  
- Procesos entre sistemas internos (caja, base de datos, impresoras, ATM)

Mi enfoque se basa en garantizar la calidad del producto, la estabilidad de los sistemas y la óptima experiencia del usuario final.

Cuento adicionalmente con conocimientos en **automatización de pruebas** (Selenium con Java, Cypress) y en **testing de APIs** (Postman, Karate), conocimientos que estoy ampliando de forma continua.

---

## ¿Qué contiene este portafolio?

Este repositorio representa un flujo de trabajo completo de QA aplicado a una aplicación web seleccionada para fines demostrativos.

| Carpeta | Descripción |
|--------|-------------|
| **01-plan-de-pruebas** | Documento con alcance, objetivos, criterios de entrada/salida, riesgos y estrategia de pruebas. |
| **02-casos-de-prueba** | Casos de prueba estructurados según requerimientos funcionales y criterios de aceptación. |
| **03-matriz-de-trazabilidad** | Relación entre requisitos, casos de prueba, evidencias y defectos. |
| **04-reportes-de-bugs** | Defectos documentados con detalle, severidad y pasos para reproducir. |
| **05-testing-exploratorio** | Sesiones exploratorias con notas, heurísticas y hallazgos. |
| **06-evidencias** | Capturas de pantalla, videos y evidencias de ejecución. |

---

##  Herramientas y tecnologías utilizadas

- **Front-end Testing:** Selenium con Java, Cypress  
- **API Testing:** Postman, Karate  
- **Gestión de pruebas:** JIRA, Xray, ALM, AIO Test  
- **Gestión de CI/CD:** Jenkins, Maven  
- **Documentación y seguimiento:** Excel, Office, GitHub  

---

## Metodologías aplicadas en este portafolio

- Análisis de requerimientos  
- Diseño de plan de pruebas  
- Creación de casos de prueba basados en criterios de aceptación  
- Matriz de trazabilidad  
- Testing funcional manual  
- Testing exploratorio  
- Reporte y seguimiento de defectos  
- Ejecución y documentación de evidencias  

---

## Objetivo de este portafolio

Este portafolio tiene como propósito demostrar:

- Mi capacidad para estructurar documentación QA de principio a fin  
- Mi forma de trabajar con casos de prueba, defectos y trazabilidad  
- Mi pensamiento crítico en la detección y análisis de errores  
- Mi disciplina en la documentación de evidencias  
- Mi orientación a la calidad y mejora continua
